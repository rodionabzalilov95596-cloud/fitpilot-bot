## ФитПилот

Бот‑фитнес‑тренер и планировщик питания с анкетой, кнопками‑меню, хранением профиля пользователя и генерацией планов через LLM (по умолчанию — провайдер YandexGPT через HTTP API).

### Что уже есть
- Диалоговый движок со **сценариями** (анкета → план тренировок → план питания → чек‑ины).
- Хранение состояния пользователя в JSON (подходит для MVP).
- HTTP сервер с webhook‑эндпоинтом и “эмулятором” сообщений для локального теста.
- Абстракция LLM‑клиента + провайдер `yandex` (настройка через `.env`).

### Как запустить бота (пошагово, Windows)

**Сайт** (`site/index.html`) и **бот** — это разное. Сайт открывается двойным кликом. Бот запускается через Node.js в терминале.

#### Шаг 1 — установить Node.js (один раз)

1. Откройте https://nodejs.org
2. Скачайте **LTS** и установите (всё по умолчанию, галочка «Add to PATH»)
3. **Перезапустите Cursor** (или откройте новый терминал)

Проверка в терминале:

```powershell
node --version
npm --version
```

Обе команды должны показать номер версии.

#### Шаг 2 — токен и ID сообщества ВК

1. Токен — в `.env.txt` (строка `vk1....`) или в `.env` как `VK_BOT_TOKEN=...`
2. **Обязательно** укажите ID сообщества в файле `.env`:

```env
VK_GROUP_ID=123456789
```

Число берётся из адреса группы: `vk.com/club123456789` → `VK_GROUP_ID=123456789`  
(или `public123456789` — то же число).

Токен должен быть **ключом доступа сообщества** (Управление → Работа с API), не личным токеном.

#### Шаг 3 — запуск

**Вариант А (проще):** дважды кликните `start.bat` в папке проекта.

**Вариант Б (терминал в Cursor):** `Terminal` → `New Terminal`, затем:

```powershell
cd C:\Users\USER\Desktop\bot
npm install
npm run dev
```

#### Шаг 4 — проверка

В терминале должно быть:

```
FitPilot listening on http://localhost:3000
VK long poll started (desktop testing mode)
VK long poll: group_id=...
```

Окно терминала **не закрывайте** — пока оно открыто, бот слушает ВК.

Напишите **в личные сообщения** вашему сообществу ВК (кнопка «Написать сообществу»). Бот ответит приветствием.

#### Остановка

В терминале нажмите `Ctrl+C`.

### Как тестировать без “Макса”
Сервер поднимает эндпоинт:
- `POST /simulate` — имитация входящего сообщения от пользователя (включая “нажатие кнопки”)

Пример тела запроса:

```json
{
  "userId": "u1",
  "text": "Привет"
}
```

### Подключение ИИ

Бот генерирует план тренировок и питания через LLM после анкеты.

**Вариант 1 — OpenAI** (проще всего, подходит OpenRouter / ProxyAPI):

```env
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-ваш-ключ
OPENAI_MODEL=gpt-4o-mini
```

Если используете другой хост API — добавьте `OPENAI_BASE_URL=https://...`.

**Вариант 2 — YandexGPT:**

```env
LLM_PROVIDER=yandex
YANDEX_IAM_TOKEN=...
YANDEX_FOLDER_ID=...
YANDEX_MODEL_URI=gpt://.../yandexgpt/latest
```

**Для тестов без ИИ:** `LLM_PROVIDER=mock` (заглушка с примером плана).

После смены настроек перезапустите бота (`Ctrl+C` → `npm run dev`).

### Бот ВКонтакте (long polling)
Пока бот в MAX недоступен, используется адаптер VK с long polling — удобно тестировать с десктопа.

1) Положите токен сообщества в `.env` (`VK_BOT_TOKEN=vk1....`) или просто строку `vk1....` в `.env.txt`.
2) В настройках сообщества ВК включите **Сообщения** и **Bots Long Poll API** (событие `message_new`).
3) Запустите `npm run dev` — бот начнёт слушать входящие сообщения.

Подробнее: `src/adapters/vk/README.md`.

### Бот MAX + календарь работы (miniapp)

В анкете после вопроса о поле/цикле пользователь настраивает **график работы** в мини-приложении-календаре. Команда `/grafik` в MAX открывает редактор в любой момент.

Miniapp требует **постоянный HTTPS** — для этого используется деплой на [Render](https://render.com) (бесплатный тариф).

#### Деплой на Render (пошагово)

1. Залейте проект на GitHub (или подключите репозиторий к Render).
2. На [render.com](https://render.com): **New → Web Service** → выберите репозиторий.
3. Настройки:
   - **Build Command:** `npm install && npm run build`
   - **Start Command:** `npm start`
   - **Health Check Path:** `/health`
4. В **Environment** добавьте переменные (как в `.env`):
   - `MAX_BOT_TOKEN`
   - `LLM_PROVIDER=yandex`
   - `YANDEX_API_KEY`, `YANDEX_FOLDER_ID`, `YANDEX_MODEL_URI`
   - `VK_BOT_TOKEN`, `VK_GROUP_ID` (если нужен VK)
5. После деплоя Render выдаст URL вида `https://fitpilot-bot.onrender.com`. Он подставится в `RENDER_EXTERNAL_URL` автоматически.
6. В [MAX для партнёров](https://business.max.ru): **Чат-боты → ваш бот → Расширенные настройки** → URL мини-приложения:
   ```
   https://ВАШ-СЕРВИС.onrender.com/miniapp/schedule/
   ```
7. Перезапустите сервис на Render (или дождитесь деплоя). В логах будет строка `Miniapp URL`.

**Локально** (`start.bat`) бот и VK работают как раньше. Календарь из MAX на телефоне откроется только после деплоя на Render.

> На бесплатном тарифе Render сервис «засыпает» без трафика (~1 мин на первый ответ). Данные пользователей в `data/` могут сброситься при пересборке — для продакшена позже подключите диск или БД.

Точка входа адаптера MAX: `src/adapters/max/README.md`.
